# تجهيز المشروع لـ Railway و GitHub

## قبل الرفع على GitHub
- تأكد أن ملفات البيانات ليست داخل المستودع.
- هذا المشروع أصبح يحفظ قاعدة البيانات والملفات المرفوعة داخل `APP_DATA_DIR`.
- لو لم تضبط المتغير، سيستخدم:
  - محليًا: `./data`
  - على Railway: `/app/data`

## خطوات الربط مع Railway
1. ارفع المشروع على GitHub.
2. أنشئ Service جديدة على Railway من GitHub Repo.
3. أضف Volume واربطه على المسار `/app/data`.
4. أضف Environment Variable باسم `APP_DATA_DIR` وقيمته `/app/data`.
5. بوسطة: إمّا تضيف `BOSTA_API_KEY` داخل Environment Variables أو تترك الملف `bosta_api_key.txt` داخل جذر المشروع كما هو.
6. اعمل Deploy.

## النسخ الاحتياطي
- من داخل لوحة التحكم:
  - `Export البيانات`: يخرج ملف zip يحتوي على قاعدة البيانات والملفات المرفوعة.
  - `Import البيانات`: يعيد استيراد نسخة zip أو db.

## ملاحظات
- زر Export/Import متاح للـ admin فقط.
- الأوردر المسحوب من مخزن الشنط الجاهزة يظهر بعلامة واضحة داخل صفحة الأوردرات.
